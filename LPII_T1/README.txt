Trabalho Prático: Locks e Threads

Todo o código-fonte do programa está escrito em dois arquivos, por questões de organização. Temos então 'main.cpp' e 'functions.hpp', ambos no diretório principal.

Escrevi um arquivo Makefile para tornar a compilação mais prática, assim, para compilar o programa basta executar o comando '$ make all' no diretório raiz do programa (onde 'main.cpp' está). Há outra opção para o Makefile: é possível usar o camando '$ make clean'. Isso vai apagar o executável, os arquivos de resposta gerados pelo programa e qualquer arquivo temporário que surgir.

Uma vez compilado, pode-se executar o programa com o comando '$ ./lp2-t1'.

Assim como estabelecido para o trabalho, existem 3 subpastas, as quais uma delas contém as requisições que serão lidas pelo programa, outra os arquivos que poderão ser processados e outra para os arquivos de resposta gerados pelo programa.

Existem 14 arquivos de requisição (que devem gerar 14 respostas equivalentes) e 9 arquivos de entrada escolhidos por mim, dos quais 4 são imagens .jpg e 5 são arquivos de texto.
